First Steps

Not sure how to begin? Not sure if you can?

Relax. This section will cover the basics of the basics, ensuring that you don't get lost.

This section will not teach you how to code, or how to set up a basic HTML page.

This section will:

  1. Provide two options for basic webpage formats for use while following this guide
  2. Direct you to additional resources to fill in any gaps about website creation for your own research and use
  3. Direct you to the Food2Fork API
  4. Get you set up as a valid user with the Food2Fork API


My webpage needs what!?

There are many different ways to create a webpage. For the purposes of this guide, it will be assumed that you are using HTML and PHP.

HTML stands for HyperText Markup Language. It is simply a language used to create webpages.

PHP is a server-side scripting language. It allows us to do many things that straight HTML just can't do, and you can mix it right in with the HTML.

At least one of the files created will need to be a PHP file so we can receive information from the Food2Fork API. This will mean, for the purposes of following along in the tutorial, you will have one of two options:

  1. A PHP file that will be set up to receive user input, then display the requested information on the same page.
  2. An HTML file that will be set up to receive user input, and a second PHP file that will receive the requested information from the API, and display the contents to the user.

If you're an individual who is comfortable developing websites, and are currently rolling your eyes at the basic information above, that's fine! Please use the Food2Fork API guide to enhance your current website, and feel confident that your user experience will exceed the simple example provided at the end of this "How-to Guide."


At least one of those things scared me. Help.

Don't be scared. Just know you have a little bit of reading to do. There are many resources available on the web for free, but I will provide some suggetions that are easy to use, and easy to search.

To learn more about HTML, please go to: W3schools They'll teach you all about using HTML, what everything means, give you examples, and let you try out features, all for free.

If you aren't sure how to create a PHP file, I suggest using an IDE. An IDE is an "integrated development environment." What that means to you is that it's a program you'll type into that will help guide you and make the webpage development easier. You can do all sorts of cool things with it, but what those things are will vary with the IDE you pick.

This author uses a wonderful and free IDE called Sublime, which I highly recommend: Learn about or Download Sublime


I'm ready, bring on the API information!

The first step in using the API is to visit the Food2Fork API site.

Please click here to visit the site now.

The website, provided there have been no overhauls since this guide was created, should look like this:

To be able to receive data from the Food2Fork API, you will need to sign up. It's free.

Please go click on the "sign up" button.

Enter your information in the appropriate fields, and you will receive a message saying you are an API admin. Congrats! That means you now have an API Key that will give you access to the data at Food2Fork.

Your API Key will be a series of letters and numbers. For the purposes of this tutorial, my key is kept secret. This key is important, and will be used shortly when we cover connecting your website to the API.

It's worth noting that you can click the "Upgrade Plan" button if you'd like the API to no longer be free. This action will allow more requests from your website for data from the Food2Fork API per day. Currently the free limit is set to 500 requests per day. Ultimately, it is up to you to decide if that's needed. For the purposes of this tutorial, we're staying free.
