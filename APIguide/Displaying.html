Receiving and Displaying Information

Want to show recipe names? Provide the recipe URL? Tell your users how highly it's ranked?

You can do all of those things and more! You just need to understand what information the Food2Fork API provides, and how you can access that information. Once accessed, there are many ways you can display it.


What information is hiding in the variable I made?

The main page for the Food2Fork API provides a list of response parameters. Though that may look or sound intimidating, it's actually just providing you with how the information is organized, and giving you the names to use to access the information.

Let us start by looking at the response parameters:

This information basically demonstrates how you access the information.

First, you access the variable you made.
Next, you access recipes, page number, or recipe count.
Then, you have access to anything within recipes!

The parts of the Food2Fork API search you can access and display are:

  1. image_url: This provides the URL to a tasty recipe picture. It can be displayed by itself, or as the source URL for an HTML image if you'd like to display the picture.
  2. source_url: If you don't want the recipe to link to Food2Fork in any way, this is the link to use. It will direct the user to the original website the recipe was displayed on.
  3. f2f_url: If you like consistency, however, all your recipes can link back to the Food2Fork site. This will display the recipe as shown on Food2Fork.
  4. title: The name of the recipe.
  5. publisher: Don't steal, it's bad. This will tell the audience where the delicious recipe first came from.
  6. publisher_url: Instead of linking to the source url, this will simply link to the main website the recipe was originally posted on.
  7. social_rank: Food2Fork has a social ranking algorithm they deploy, checking to see which recipe is most highly talked about. It will give you a number rating.
  8. page: The page searched that the individual recipe was located at.
  9. recipe_id: This is not listed on the parameter list, but it is available and useful! This will give us a unique identification code for a recipe, which we can then send in another request to gain access to the ingredient list. How to do this will be covered shortly.

There is one important features you can access outside of "recipes", however. If you look at the above provided charts, you can also see that in line with "recipes" there is a parameter named "count." That means you wont have to access "recipes" first before getting this information:

  1. count: The total number of recipes your search will return.

What about the ingredient list?

Displaying the ingredient list is only slightly more complicated, because we don't know how many ingredients are in the list. Rest assured, however, that we can still manage. What's more, if you're feeling adventerous, you can apply this knowledge to the general search to return the maximum 30 results per page each time.

Let us start by looking at the parameters offered by Food2Fork:

As illustrated in the below chart, much of the information is simply a repeat information that we already have. However, there is one additional, new piece of information we can get : ingredients! To gain access to the ingredient information, we will follow the flow chart, just as we did above:

Studying the flow chart, we can see that we are really just doing what we've already done, but with different names.
First, you access the variable you made.
Next, you access recipe.
Then, you access the ingredients for that recipe

The only extra step we have to take is figuring out how many ingredients are stored in "ingredients," and we can do that easily with the PHP function "count."

We will make a variable, let's call it "size", and put the value the "count" function returns in count.

Then, just like with our previous loop, we will create a variable called "i", which will hold the value 0 through the max ingredient amount determined by count. After each loop, "i" will get bigger by one until it finally reaches the value of the variable "size." This ensures we print every ingredient in the list, even though we might not know how big that list is.
